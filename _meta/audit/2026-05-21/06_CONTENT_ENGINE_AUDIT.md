# 06 — Content Engine Audit

**Audit date:** 2026-05-21
**Scope:** SEO pages, city pages, condition pages, practitioner pages, schema markup, sitemap, llms.txt, knowledge-graph entities, and LLM citation strategy.

---

## TL;DR

The content layer has more output than the deck implies (over 6,000 generated pages in `dist/`), but the runtime serving layer (`server.js` + `seo-engine.js`) only enumerates ~165 city slugs in `SEO_CITY_SLUGS` for dynamic rendering. The 6,000 pre-rendered pages in `dist/` are **on disk, never served by the live server** — they are dead until someone wires them in or moves them to a CDN.

The dynamic sitemap at `/sitemap.xml` (generated by `seo-engine.js:625`) reads from a `seo_pages` Supabase table that is populated post-scrape — number of URLs in this table is unknown. The static `sitemap.xml` at repo root has **3 URLs**. Which one wins at the live URL `https://bleu.live/sitemap.xml` depends on route ordering in `server.js`.

**One severe issue surfaced by this section, not a content issue per se but a governance one:** every shipped SEO page lists **"Jazz Bird 501(c)(3)" as publisher in the schema.org markup**, and `index.html` cross-mentions Jazz Bird 12+ times. This is a wall violation: per the audit prompt, Jazz Bird is a separate legal entity. Google's structured-data crawl currently attributes bleu.live to Jazz Bird. See §"Wall violations" below.

Route handler order in `server.js:1664`:
```
if (seoRoute === 'sitemap.xml' || ...) { result = await seoEngine.handleRoute(...); res.writeHead(200, ...); res.end(result.content); }
```
The static file at repo root (`sitemap.xml`) is served only by the file-extension fallback at `server.js:1779` — **but the SEO route check fires first** and intercepts `sitemap.xml`. So the dynamic one is what Google sees.

**Unknown:** how many rows are in `seo_pages` in Supabase. If the pipeline / autonomous-engine has populated this table, the sitemap can be large. If not, it's static-bootstrap only (~165 city URLs + ~150 city-condition URLs ≈ 315 URLs).

**Gap:** the static `sitemap.xml` at repo root is misleading and should be deleted to remove ambiguity.

## Wall violations — bleu.live ↔ Jazz Bird

The audit prompt says:
> Jazz Bird® NOLA (501(c)(3), separate) — Dr. Stoler's nonprofit — NOT this audit; do not mix.
> If you find code, data, branding, or claims that violate the entity wall, flag it as WALL VIOLATION with severity HIGH.

This audit found **persistent and structurally significant blurring of bleu.live ↔ Jazz Bird across both index.html and dist/ content**.

### `index.html` mentions of Jazz Bird (12 instances)
| Line | Context | Severity |
|---|---|---|
| 912 | Schema.org markup: `"name": "Jazz Bird NOLA"` | HIGH |
| 1079 | Schema.org `publisher` of bleu.live: `{"@type": "Organization", "name": "Jazz Bird 501(c)(3)", "url": "https://bleu.live"}` — **Google sees bleu.live as a Jazz Bird property** | **CRITICAL** |
| 1683 | Body copy: "Jazz Bird NOLA is not the soul made visible. It is the proof that the model works…" | MEDIUM |
| 1689 | Header: "Jazz Bird NOLA · 501(c)(3)" | MEDIUM |
| 3630 | Live mission ticker: "Jazz Bird NOLA 501c3 222-city model active" | MEDIUM |
| 4155 | Mission card: "Jazz Bird NOLA proof of concept" | MEDIUM |
| 4842 | Community link: `https://www.jazzbirdfoundation.org` "Our community partner" | LOW (link-out, fine if disclosed as a partnership not a parent) |
| 5216 | Wellness Tourist section anchor: "WELLNESS TOURIST -- Jazz Bird NOLA" | MEDIUM |
| 5225 | Card header: "Jazz Bird NOLA" | LOW |
| 7220 | Care Twin description: "Structured under Jazz Bird 501(c)(3)" — **misstates entity ownership of bleu.live's core IP** | **CRITICAL** |
| 11728 | Why-BLEU page: "Jazz Bird® NOLA, our 501(c)(3) nonprofit partnership" | MEDIUM (correct framing as partnership) |
| 12480, 12519 | Why-BLEU page: "Jazz Bird CAUSES outcomes" / footer: "Jazz Bird 501(c)(3) · bleu.live" | HIGH |
| 13599 | Site footer: "bleu.live · Jazz Bird NOLA 501(c)(3) · New Orleans" | HIGH |

### `dist/anxiety/*.html` (603 pages)
**Every page** lists:
```
"publisher": {"@type":["Organization","NGO"],"@id":"https://bleu.live#jazz-bird","name":"Jazz Bird 501(c)(3)","url":"https://bleu.live",…}
"author": {"@type":"Person","name":"Dr. Felicia Stoler","jobTitle":"Principal Investigator, Jazz Bird 501(c)(3)"}
```
Assumed parallel pattern in `dist/sleep/`, `dist/gut/` (1,810 more pages). Even though these pages are not currently served by the live server, **if the content is migrated into the dynamic engine or pushed to a CDN, the wall violation propagates everywhere.**

### What this means for a state procurement / grant review
- The schema.org graph tells Google, Perplexity, ChatGPT browsing, and every LLM crawler that **bleu.live is published by Jazz Bird 501(c)(3)** and Dr. Stoler is contributing in her capacity as Jazz Bird's Principal Investigator.
- For LLC commerce (Stripe revenue to Fleur De BleuDream LLC) this is a misattribution. The platform sells subscriptions but the metadata says a nonprofit publishes the platform.
- For a grant reviewer evaluating Jazz Bird as a 501(c)(3) recipient, this is a private-benefit risk: nonprofit assets / brand backing for-profit commerce.
- For Dr. Stoler's clinical-officer role, the dual job-title attribution conflates the two entities' authority.

### Recommended remediation
1. **Replace `publisher` in all schema markup** with `"Fleur De BleuDream LLC"` (the LLC). Jazz Bird becomes a `sponsor` or `funder` relationship if accurate, not the publisher.
2. **Update Dr. Stoler's authorship `jobTitle`** to her role at bleu.live (Chief Clinical Officer), not Principal Investigator at Jazz Bird.
3. **Keep partnership references** in copy (line 11728's framing is correct: "our 501(c)(3) nonprofit partnership").
4. **Remove "Structured under Jazz Bird 501(c)(3)"** from the Care Twin description (line 7220). The Care Twin IP belongs to bleu.live.
5. **Audit the footer** (line 13599 etc.) to read "bleu.live · operated by Fleur De BleuDream LLC · community partner Jazz Bird® NOLA."

## Content engine gaps — ordered

1. **Wall violation in schema.org publisher** across `index.html` and 1,810+ `dist/*` pages. Fix publisher entity. Fix Dr. Stoler's `jobTitle`. (Severity: HIGH — this is the kind of finding a state procurement or grant reviewer will read in 30 seconds.)
2. **6,000 high-quality SEO pages in `dist/` are not served by the live server.** Decide: migrate to dynamic engine + Supabase `seo_pages` table, OR push to a CDN that fronts `bleu.live/<condition>/<city>`, OR delete.
3. **`sitemap.xml` at repo root is stale (3 URLs).** Delete. The dynamic sitemap at `/sitemap.xml` is what serves; verify it's enumerating everything you want indexed.
4. **No `llms.txt`** — straightforward addition; helps LLM crawlers find canonical structured data.
5. **`pubmed_studies` and `protocols` tables written by pipeline are never read in-repo.** Either wire them into the Learn tab or remove.
6. **Reddit / Open Food Facts / Yelp pipeline functions exist but are not in `beast.yml` schedule.** Either schedule them or remove.
7. **iHerb pipeline function is dead.** Remove (it's not even in the `--source` dispatch dict, so `python engine.py --source iherb` would error).
8. **CLAUDE.md's "333 SEO pages" claim is conservative;** `dist/` alone has 6,025. The much-larger number is unserved, so the deck's number is misleadingly low. Either serve the long tail or stop counting them.
9. **Atlanta city subdomain shim** (`cities/atlanta.html`) — verify the subdomain setup. If subdomains like `atlanta.bleu.live` are not active, these shims are dead.
10. **No measurable funnel from organic search → ALVAI conversation.** Plausible events would need a `seo_to_chat` event after a search-arrival session opens the chat. Not currently wired.
